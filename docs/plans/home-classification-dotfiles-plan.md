## Summary

Fold three additive capabilities into `bin/cloud-xdg-provision.sh` (home-tree.sh stays frozen ‚ÄĒ user will never use it): (1) a **classification taxonomy** for every top-level `~/` dir via a **separate** `HOME_CLASS_REGISTRY` table in `bin/lib/xdg-common.sh` (the XDG-symlink lane stays byte-identical except the user-confirmed P2 move of `Projects`‚ÜíCODE); (2) **rclone-remote offload-on-demand** for git-managed code dirs (`repos`, `AndroidStudioProjects`, `Projects`) ‚ÄĒ push to a cloud remote, verify with read-back, then drop local behind a 5-guard git gate, with `--hydrate` to restore; (3) a **dotfiles bare-repo** mode (`--dotfiles-init/-track/-status`) with an idempotent sourced alias file. The offload lane uses an **rclone REMOTE** (the only model that frees local space) and is mutually exclusive with the existing symlink-into-mount lane via mode dispatch. **User decisions (2026-06-30):** D2=P2 (Projects‚ÜíCODE, `~/Projects` symlink migrates, L3 goldens 9‚Üí8; Desktop/Documents stay offloaded); machine-local = `pyenv/Applications/log/QEMU`; v1 = git repos only (QEMU/data dirs ‚Üí v2).

## Specialist Perspectives

### ūüďč Preparation Phase (preparer, Task #10 ‚ÄĒ complete)
**Effort**: Medium. Artifact: `docs/preparation/research-offload-dotfiles-macos.md`

Key findings (all resolved, no open research blocking design):
- **Only an rclone REMOTE frees local space.** The mounted-cloud-path model (cloud-xdg's current lane) frees nothing until the provider evicts ‚ÄĒ so offload-on-demand must use the remote model (home-tree's lane).
- **Safe offload sequence**: `rclone copy --immutable` ‚Üí `rclone check --download --one-way` (read-back proves *durable* upload, stronger than a mount-view stat) ‚Üí drop local **only** after a 5-guard git gate.
- **Venvs are unsafe to offload** (absolute paths in `bin/activate`, shebangs, `pyvenv.cfg`) ‚Üí justifies `Pyenv` as machine-LOCAL class; recreate from `requirements.txt`.
- **Dotfiles**: bare `$HOME/.dotfiles`, `status.showUntrackedFiles no`, dedicated sourced `~/.config/xdg-cloud/aliases.sh`.
- **iCloud-only machines**: `brctl evict`/`download` as a separate macOS+iCloud+Optimize-Storage-gated sub-mode, not default.

#### Design Approach ‚ÄĒ two mutually-exclusive lanes
- **Lane 1 (default, unchanged)**: symlink-into-mount provisioning. CODE-class dirs **never** enter `OFFLOAD_SET`.
- **Lane 2 (new)**: rclone-remote offload-on-demand, per-git-repo, frees space. Bypasses `ensure_cloud_tree`/redirect loop entirely.
- Mode dispatch after arg parsing routes each invocation to exactly one lane.

#### Offload safety gate (G1‚ÄďG5, fail-closed, PER-GIT-REPO)
- **G1** is-git-repo: `git -C "$d" rev-parse --is-inside-work-tree`
- **G2** clean tree: `[ -z "$(git -C "$d" status --porcelain)" ]`
- **G3** fully pushed: every local branch has an upstream AND `rev-list --count @{u}..HEAD == 0` (refuse if any branch lacks upstream or is ahead)
- **G4** no stashes: `[ -z "$(git -C "$d" stash list)" ]`
- **G5** cloud verified: `rclone check --download --one-way` rc=0

Containers hold **many** repos ‚Üí guards run per-repo, not per-container. Untracked/ignored-but-precious files ‚Üí warn (rclone copy is their only backstop).

#### Testing discipline (critical)
- **Two layers**: real rclone ‚Üí local remote for happy path (a stub that always returns 0 hides real verify/filter bugs); PATH-shim rclone only for failure injection on the gate.
- **Destructive 'drop local' is P0 data-loss surface**: always sandbox-HOME with an in-sandbox target guard; never against real `$HOME`.
- **Registry is read POSITIONALLY**: append new columns at the END; keep `CLOUDXDG_KEYS`/`HOMETREE_KEYS` explicit so L3 + ¬ß6/¬ß7 goldens stay green. Any membership/order change = deliberate same-commit golden update, never silent drift.
