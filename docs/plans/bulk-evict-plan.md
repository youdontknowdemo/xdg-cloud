# Implementation Plan: Bulk Skip-and-Report Evict

> Generated by `/PACT:plan-mode` on 2026-07-07
> Status: IN_PROGRESS

## Summary

Change `--icloud-evict` from all-or-nothing (refuse the whole set if any candidate
isn't fully uploaded — the `.DS_Store` wedge that makes directory evict unusable) to
**evict only the individually-proven-`uploaded` subset, skip-and-report the rest**. This
is a semantic change to a **data-destructive** lane, so the design centers on one hard
invariant: **no file is ever passed to `brctl evict` without individual proof it is
`uploaded`.** Four consultants (preparer, architect, security, test) converged; the one
architectural conflict — how evict-target paths are derived — is resolved below in
security's favor (order-join, never path-extraction from helper stdout).

**User decision (2026-07-07):** skip-and-report becomes the **DEFAULT** of `--icloud-evict`
(no opt-in flag). Rationale: the per-file destruction predicate gets *stronger*, not weaker;
an opt-in flag would make TUI-evict diverge from CLI-evict or leave the default lane wedged.
The cost is a **backward-incompatible exit-code change** (a mixed set now exits 0/reports
instead of exiting 1/refusing) — called out loudly in CHANGELOG; the "is everything
uploaded?" probe moves to `--icloud-status`/`--icloud-sync-status`.

## The safety design (resolved)

### Join model — ORDER-JOIN, never path-extraction (resolved MAJOR conflict)

The architect proposed deriving the evict path *from* each helper record (`uploaded\t<path>`).
Security **disqualified** that for a destructive lane: any helper compromise/bug or adversarial
filename interaction that corrupts the parse would then evict an attacker-influenced path.
**Resolution (security's position, bridged with the architect's framing):**

- Evict targets come **exclusively from the `candidates[]` array** built by
  `find "$ICLOUD_TARGET" -type f -print0` (physical, absolute, confined at walk time). Helper
  stdout supplies **state only**, joined to `candidates[i]` **by argv order** (the helper emits
  one NUL record per argv path in argv order — verified in `icloud-uploaded.swift`).
- **Path-echo assertion** (the bridge): parse the record's own path field
  (`${rec#*"$tab"}`) and require it **byte-equals `candidates[i]`**, else `die`. This catches
  any desync/substitution while keeping the evict target sourced from `find`, never from a
  parsed string.
- **Count match**: records parsed must equal candidates in that chunk, **both directions**
  (surplus = contract violation / wrong `$ICLOUD_HELPER`), else `die` evict-nothing.
- NUL parsing via tempfile + `while IFS= read -r -d ''` (never `$()` — strips NUL; never
  tail-salvage `|| [ -n "$rec" ]`). Whole-record safe-match `case "$rec" in uploaded"$tab"*)`.

### Two-phase gate — the exit code survives, relocated to the exact subset

Prior reviews trusted the helper **exit code** (0 iff ALL uploaded). Under subset semantics a
non-zero batch exit is *normal*, so:

- **Phase 1** (dry-run + apply): batched, chunked helper exec → parse records → order-join with
  path-echo + count-match → build `evict_set` from `candidates[]` + the skip report. The batch
  rc is **ignored** (`|| true`) EXCEPT the one structural abort: **candidates > 0 and zero
  records parsed → die, evict nothing** ("helper produced no output — refusing to evict blind").
- **Phase 2** (apply only, per chunk, immediately before that chunk's evict loop): **re-exec the
  helper over exactly the evict subset and require rc == 0.** This re-applies the audited
  all-or-nothing exit-code gate over precisely the set being destroyed — the reviewer-trusted
  "no `brctl evict` except downstream of a helper rc-0 covering those files" property is
  preserved verbatim, and the phase-1 parser is demoted to non-safety-critical (a parse bug can
  then only mis-*select* → under-evict, never over-evict). Phase-2 rc≠0 → refuse that chunk
  fail-closed, report "state changed since plan, re-run".

### Per-file pre-evict guard (TOCTOU residue)

iCloud sync concurrently rewrites the tree; an app may auto-save mid-run. Immediately before
each `brctl evict` (apply): re-confine the path lexically under the resolved root, and `lstat`
it — still a **regular file, not a symlink, not dataless**; skip+warn on any drift. Whether
`brctl evict` refuses a dirty file is undocumented — a PREPARE go/no-go probe settles it; the
guard + consent flag cover the residual either way.

### Fail-closed direction (all ambiguities)

Helper missing → die (unchanged gate). Exec rc 2/≥126/signal → die. Unparseable record → that
file skipped; *all* unparseable → die. Truncated final record → dropped. Record count ≠
candidate count → die. Empty output on non-empty candidates → die. Zero uploaded → no-op return
0. Phase-1/phase-2 disagreement → die. Post-selection file change → skip+warn.

#### Key Decisions (resolved)
| Decision | Ruling | Rationale |
|----------|--------|-----------|
| Default vs opt-in flag | **Default change** (user-approved) | Predicate strictly stronger; opt-in causes TUI/CLI drift |
| Join model | **Order-join + path-echo assertion + count-match** | Evict target always from `find`; record supplies state; echo catches desync (security) |
| Exit-code gate | **Two-phase; phase-2 rc==0 over the subset per chunk** | Preserves the audited gate over the exact destroyed set (all three safety consultants) |
| Per-file guard | **lstat (regular/not-symlink/not-dataless) + re-confine before each evict** | TOCTOU residue from concurrent iCloud sync (security #8) |
| rc taxonomy | rc 0 = completed sweep (incl. all-skipped); rc 1 = gate failures + zero-records abort | Keeps TUI `interpret_result` wiring at zero change |
| Report | **Verbatim, no porcelain**; skip listing capped 20/reason + a machine-stable per-reason summary count line | Standing ruling (iCloud panes verbatim); stable line gives smoke a grep surface |
| Consent | `--i-understand-data-loss-risk` + TUI typed-path gate **unchanged**; skip report **mandatory before any evict**, apply AND dry-run | Optimize-Mac-Storage risk is per-file, unchanged; report prevents model drift (security #4) |

#### Report format (target)
```
iCloud evict under: <resolved> (evicts ONLY individually-proven-uploaded files; others skipped)
  skipped (NEVER evicted — fail-closed):
    not-uploaded    /path/a
    not-in-icloud   /path/.DS_Store
    … and N more not-uploaded (see '--icloud-status <path>')
evict plan: 42 of 57 candidate file(s) proven uploaded — 1.9 GiB potential free*
skipped: 15 file(s) — not-uploaded: 3, not-in-icloud: 11, helper-error: 1, unanswered: 0
*potential: with 'Optimize Mac Storage' OFF, evict frees nothing (no programmatic check).
[dry-run] would evict the 42 proven-uploaded file(s) above (15 skipped). Re-run with --apply.
```

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Parser bug evicts an unproven file | Low | Critical | Order-join (target from `find`, not stdout) + phase-2 rc==0 re-gate + path-echo + count-match; mutation-verified P0 tests |
| TOCTOU: file dirtied/swapped between check and evict | Medium | High | Phase-2 narrows window; lstat pre-evict guard; consent flag covers residual; PREPARE probe on brctl-dirty behavior |
| Exit-code change breaks external scripts | Medium | Medium | CHANGELOG loud callout; probe use moves to `--icloud-status` |
| Report understates/overstates what was evicted → operator model drift | Low | Medium | Mandatory skip report before any evict (apply + dry-run); final message states both counts |
| Newline/tab filename forgery | Low | Critical | NUL parse + order-join make forgery inert (already fixed class, v0.4.1); dedicated adversarial P0 tests |
