## 4. Managed-lane probe (NEW standalone helper — does NOT modify `dotfiles_guard_path`)

`dotfiles_guard_path` (merged) *dies*; adopt needs a *boolean*. Add a standalone predicate (same
name-based top-component check; ends with explicit `return 1` to dodge the set-e trailing-`&&`
footgun). Keeping it separate leaves the merged track path untouched.

```sh
# True (0) if the top-level $HOME component of a repo-relative path is a cloud-xdg-managed lane
# (CLOUDXDG redirect target / CODE container / LOCAL dir) or the bare repo itself. No die.
dotfiles_path_is_managed() {
  local rel="$1" top k nm
  rel="${rel#/}"; top="${rel%%/*}"
  [ "$top" = ".dotfiles" ] && return 0
  for k in $CLOUDXDG_KEYS; do                                  # shellcheck disable=SC2086
    nm="$(local_name "$(field "$(registry_row "$k")" 2)" "$(field "$(registry_row "$k")" 3)")"
    [ "$top" = "$nm" ] && return 0
  done
  for k in $CODE_KEYS; do                                      # shellcheck disable=SC2086
    nm="$(local_name "$(field "$(code_row "$k")" 2)" "$(field "$(code_row "$k")" 3)")"
    [ "$top" = "$nm" ] && return 0
  done
  for k in $LOCAL_KEYS; do                                     # shellcheck disable=SC2086
    nm="$(local_name "$(field "$(code_row "$k")" 2)" "$(field "$(code_row "$k")" 3)")"
    [ "$top" = "$nm" ] && return 0
  done
  return 1
}
```

## 5. `cmd_dotfiles_adopt` — the flow

```sh
cmd_dotfiles_adopt() {                        # $1 = repo URL
  begin_mutating_mode                          # guard_not_root + trap + (apply-only) lock
  local url="$1" tracked p abs bak stamp n aside_list tmp
  # ---- refuse-clobber: adopt is fresh-machine ----
  if [ -e "$DOTFILES_DIR" ] || [ -L "$DOTFILES_DIR" ]; then
    if dotfiles_is_ours; then
      die "$DOTFILES_DIR already exists (already initialized). Adopt is for a FRESH machine —
  use 'dotfiles pull' / --dotfiles-track for an existing repo."
    fi
    die "$DOTFILES_DIR exists but is not our bare repo — refusing to clobber it."
  fi
  dotfiles_resolve_rc                          # sets RC_TARGET or dies IN THIS SHELL (before any mutation)

  # ---- dry-run: temp-clone preview (nothing under $HOME is touched) ----
  if [ "$DRY_RUN" -eq 1 ]; then
    info "[dry-run] git clone --bare '$url' -> $DOTFILES_DIR ; config status.showUntrackedFiles=no"
    tmp="$(mktemp -d "${TMPDIR:-/tmp}/xdg-adopt.XXXXXX")" || die "cannot create temp dir for dry-run preview"
    if git clone --bare "$url" "$tmp" >/dev/null 2>&1; then
      tracked="$(git --git-dir="$tmp" ls-tree -r --name-only HEAD 2>/dev/null || true)"
      while IFS= read -r p; do
        [ -z "$p" ] && continue
        if dotfiles_path_is_managed "$p"; then info "[dry-run] WOULD REFUSE: repo tracks managed path '$p'"; continue; fi
        if [ -e "$HOME/$p" ] || [ -L "$HOME/$p" ]; then info "[dry-run] would move aside: \$HOME/$p"; fi
      done <<EOF
$tracked
EOF
      info "[dry-run] would then: dotfiles checkout (NO --force); write $DOTFILES_ALIASES + rc block on $RC_TARGET."
    else
      warn "[dry-run] clone preview failed for '$url' (network/URL?). Under --apply this aborts cleanly, \$HOME untouched."
    fi
    rm -rf "$tmp"
    return 0
  fi

  # ---- CLONE (apply). Failure => remove the partial repo, $HOME untouched. The trap window is
  #      NOT armed here: clone (and the managed pre-scan below) touch NO user data — only the fresh
  #      bare repo — and both clean up explicitly on failure. Arming DOTFILES_ADOPT_ACTIVE during a
  #      window that ends in a deliberate `die` would make the EXIT trap print a SPURIOUS
  #      "INTERRUPTED" message. The flag is armed ONLY around aside+checkout (where user data moves). ----
  if ! git clone --bare "$url" "$DOTFILES_DIR"; then
    rm -rf "$DOTFILES_DIR"                      # only ours-this-run (a pre-existing repo was refused above)
    die "git clone --bare failed for '$url'. Removed the partial $DOTFILES_DIR; your \$HOME is untouched."
  fi
  dotfiles_git config --local status.showUntrackedFiles no

  # ---- enumerate tracked paths (into a var; no die inside $()) ----
  tracked="$(dotfiles_git ls-tree -r --name-only HEAD 2>/dev/null || true)"

  # ---- MANAGED-LANE PRE-SCAN (fail-closed, BEFORE any aside/mutation) ----
  # Collect EVERY tracked path that hits a cloud-xdg-managed lane; if any, remove the fresh clone
  # so $HOME is FULLY untouched, then die naming ALL offenders. Uses the boolean probe (not the
  # dying dotfiles_guard_path) so it can report every offender in ONE message. here-doc fed so
  # `managed` accumulates + `die` runs in the PARENT shell. Explicit `if` (no trailing `&&`) for set -e.
  local managed=""
  while IFS= read -r p; do
    [ -z "$p" ] && continue
    if dotfiles_path_is_managed "$p"; then managed="${managed}    $p
"; fi
  done <<EOF
$tracked
EOF
  if [ -n "$managed" ]; then
    rm -rf "$DOTFILES_DIR"
    die "the remote repo tracks cloud-xdg-managed path(s) — refusing adopt (they would fight the
  symlink/offload lanes). Removed the fresh clone; your \$HOME is UNTOUCHED. Offenders:
$managed  Untrack these in the repo (or manage them via the symlink/offload lanes), then retry."
  fi

  # ---- COLLISION PRE-ASIDE (here-doc fed, NOT a pipe, so aside_list + trap flag reach the parent).
  #      Managed paths were already refused above, so this loop only asides genuine colliders. ----
  DOTFILES_ADOPT_ACTIVE=1; DOTFILES_ADOPT_STAGE="aside"; aside_list=""
  while IFS= read -r p; do
    [ -z "$p" ] && continue
    abs="$HOME/$p"
    if [ -e "$abs" ] || [ -L "$abs" ]; then
      stamp="$(date +%Y%m%d-%H%M%S)"; bak="${abs}.pre-dotfiles-${stamp}"; n=1
      while [ -e "$bak" ] || [ -L "$bak" ]; do bak="${abs}.pre-dotfiles-${stamp}.${n}"; n=$((n + 1)); done
      mkdir -p "$(dirname "$abs")"
      mv "$abs" "$bak"
      aside_list="${aside_list}    $abs  ->  $bak
"
    fi
  done <<EOF
$tracked
EOF

  # ---- CHECKOUT (NO -f/--force). Colliders are aside, so it succeeds cleanly. ----
  DOTFILES_ADOPT_STAGE="checkout"
  if ! dotfiles_git checkout; then
    DOTFILES_ADOPT_ACTIVE=0
    warn "dotfiles checkout FAILED unexpectedly after pre-asiding colliders — NOT forcing."
    [ -n "$aside_list" ] && { warn "Files moved aside (RETAINED):"; printf '%s' "$aside_list" >&2; }
    die "adopt incomplete: bare repo left at $DOTFILES_DIR; asides retained. Inspect, then retry or
  restore. Delete NOTHING until verified."
  fi
  DOTFILES_ADOPT_ACTIVE=0

  # ---- RC wiring (reuse init's writers; unchanged) ----
  dotfiles_write_aliases
  dotfiles_install_rc_source

  # ---- report ----
  if [ -n "$aside_list" ]; then
    info "Adopted '$url'. Pre-existing files were moved aside (RETAINED — reconcile, then delete):"
    printf '%s' "$aside_list"
  else
    info "Adopted '$url' (no collisions)."
  fi
  info "Open a new shell (or '. $RC_TARGET'), then: dotfiles status"
}
```

## 6. Failure matrix (enumerated)

| Failure point | State left | User-facing outcome |
|---|---|---|
| `.dotfiles` already exists (ours) | nothing changed | die: "already initialized — use pull/track" |
| `.dotfiles` exists (foreign) | nothing changed | die: "refusing to clobber" |
| `$SHELL` unknown + no `--dotfiles-rc` | nothing changed (resolve runs before clone) | die: "re-run with --dotfiles-rc" |
| **clone fails** | partial `.dotfiles` **removed**; `$HOME` untouched; no asides | die: "clone failed; partial removed; \$HOME untouched" |
| repo tracks a **managed-lane** path (pre-scan, before any aside) | fresh clone **removed**; `$HOME` **fully untouched**; NO asides | die naming **ALL** offenders: "refusing — untrack managed paths, retry" |
| **checkout fails** after asides | bare repo + asides retained; NO force | die: "adopt incomplete; asides retained; inspect" |
| **interrupt** (INT/TERM) mid clone/aside/checkout | trap prints stage-aware recovery; asides retained | re-run after `rm -rf ~/.dotfiles` if clone was partial |
| success | repo checked out; colliders in `*.pre-dotfiles-*`; alias+rc installed | info: adopted + aside list |

## 7. bash 3.2 / shellcheck / footguns (binding)

- **here-doc, not pipe**, for the aside loop — so `die` runs in the PARENT shell (a `printf | while`
  would `die` only the subshell, silently continuing) AND `aside_list` + the trap flag are in scope.
  Same rule as `main`'s relocate loop (R2).
- **`die`-inside-`$()` kills only the subshell** — tracked paths are captured with
  `tracked="$(… || true)"` (no die inside); `dotfiles_resolve_rc` already sets a global + dies in the
  parent (reused, unchanged).
- **set-e trailing-`&&`** — predicates (`dotfiles_path_is_managed`) end with explicit `return 1`;
  existence checks use explicit `if [ -e ] || [ -L ]; then …; fi` (never `A || B && C` as a bare
  statement, which returns non-zero when false and trips `set -e`).
- **No `--force`/`-f`** on checkout, ever. **Never overwrite an existing aside** (`-e` OR `-L`
  uniquifier, catching dangling symlinks). **Symlink colliders** are asided too (`mv` moves the LINK,
  not its target — safe; managed-lane symlinks are refused earlier).
- No backticks in any written string (alias/rc reuse the merged writers). No `[[ ]]`/assoc/`mapfile`/
  `<()`/`readlink -f`. `case`/keyset loops carry `# shellcheck disable=SC2086`.
- `mktemp -d` (BSD/macOS + GNU compatible) for the dry-run preview; temp removed unconditionally.

## 9. Reasoning chain (collision-enumeration + trap window)

- **ls-tree pre-aside, not error-text parsing** — *because* moving every existing collider aside
  BEFORE a `--force`-free checkout makes "never clobber" **provable and deterministic**: checkout can
  only write paths that no longer exist locally. Parsing git's "would be overwritten" text is fragile
  (localization/format) and only reacts after git refuses — the pre-aside removes the need to force at all.
- **Managed-lane refusal is a PRE-SCAN before any aside (fail-closed)** — *because* asiding a
  redirect-target symlink or a CODE container and checking repo content over it would fight the
  symlink/offload lanes. Running the check as a *separate pass over the whole tracked list before the
  first `mv`* (collect ALL offenders, remove the fresh clone, then die naming them) guarantees a
  misconfigured remote leaves `$HOME` **fully untouched** — a per-path check inside the aside loop
  could aside earlier files before reaching a managed one.
- **`DOTFILES_ADOPT_ACTIVE` armed ONLY around aside+checkout (single master handler, STAGE marker)** —
  *because* that is the only window where user data is in flux (originals moved aside, checkout
  writing). Arming it during clone or the managed pre-scan — windows that end in a deliberate `die` —
  would make the EXIT trap print a SPURIOUS "INTERRUPTED" message (`cleanup_handler` runs on every exit,
  including `die`). bash 3.2 has no trap stacking, so one flag + stage gives an accurate interrupt
  message without a second `trap`; the aside loop is here-doc-fed in the parent shell so the flag it
  sets is in the handler's scope.
- **clone-fail removes the partial, $HOME untouched** — *because* before checkout nothing under `$HOME`
  has moved, so the only cleanup is the partial bare repo; leaving `$HOME` pristine is the safe default.
- **temp-clone dry-run preview** — *because* Risk=4 warrants showing the user the EXACT files that would
  be moved aside before they commit; the preview clones to a temp dir and touches nothing under `$HOME`.
